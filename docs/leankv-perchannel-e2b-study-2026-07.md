## Baseline anchor (rule 4) and the gate

Arm A is the shipping **TurboQuant reference** codec: randomized-Hadamard rotation + one **L2-norm scale per token vector** + the N(0,1/d) Lloyd-Max codebook. It reproduces the task anchors **exactly**: E2B 2-bit softmax-KL = **0.1869** (~0.187), 3-bit = **0.0551** (~0.055). (This is a different baseline from scale_study's per-32-block amax, which is 0.298 at 2-bit; the task's anchors are the per-vector L2 path, used here.)

The raw-TQ3 **closed-loop** cliff is ~0.27. Offline attention replay understates closed-loop error ~3-5x (VQ study rule 1); the closed-loop TQ2 control was ~0.91 and TQ3 ~0.17. So the offline TQ3 anchor (**0.0551**) is the pre-image of the cliff. Two readings of the gate:

- **Literal**: 2-bit offline softmax-KL <= 0.27. Baseline A2 = 0.1869 already sits below it, so this bar does not discriminate a rescue.
- **Operative** (the one that matters): to survive CLOSED-LOOP below the 0.27 cliff, offline 2-bit must reach the TQ3 offline anchor ~0.055 (i.e. gap_closure -> 1.0). This is the bar the verdict uses.

`gap_closure = (KL_A2 - KL_arm) / (KL_A2 - KL_A3)`: fraction of A's own 2->3-bit KL gap an arm recovers while staying at 2 bits. 1.0 = as good as 3-bit; 0.0 = no better than 2-bit; negative = worse than 2-bit.

## Side information & break-even context

Per-channel side info = `16/(group*T)` bpe, cheaper than per-32-block (0.5 bpe) once `T > 32/group` (group 1 -> 32 tokens; group 8 -> 4; group 32 -> 1). So per-channel is essentially always cheaper on side-info — it loses on quality, not budget. Outliers cost `f*20` bits/element (f = kept fraction). The per-vector L2 norm of arm A costs `16/hd` (0.0625 at hd=256).

## GO / NO-GO

**Gate (stated up front):** a deployable (non-oracle, calib-fit) 2-bit-K arm reaches the operative target — offline softmax-KL at/near the TQ3 anchor (0.055), i.e. gap_closure -> 1.0 — at <= 2.5 bpe, surviving cross-corpus within 25%. (The literal <=0.27 reading is met by the baseline itself and is not discriminating; see above.)

- **Per-channel K (KIVI) alone FAILS.** Best per-channel arm `B per-chan mse g1 (2b)` = **0.2489** softmax-KL — *worse* than the per-vector Hadamard baseline (0.1869). amax per-channel is catastrophic (0.49-1.9, outlier-hijacked over the token axis); even mse_opt per-channel loses. Post-RoPE, the structured outlier CHANNELS that KIVI/KVQuant exploit are smeared across coordinate pairs by RoPE, so a per-channel scale has nothing to grab and the Hadamard rotation (which arm A keeps) does the real work. Larger channel groups are monotonically worse.
- **Dense/sparse outliers (KVQuant) are the only load-bearing mechanism, but they plateau.** The single best deployable arm is `E per-chan mse g1 + 2% outliers` = **0.1058** (gap_closure +0.616) at 2.43 bpe; global 0.0603, local 0.1172. A 2% outlier channel buys **0.143** KL on the per-channel base but only **0.048** on the per-vector Hadamard base — outliers help MORE on per-channel precisely because per-channel's error is concentrated in a few elements the sparse channel then removes exactly, whereas the Hadamard has already spread it. So the best 2-bit arm is per-channel + outliers (E), NOT per-vector + outliers (C/A = 0.1389). But even E recovers only ~62% of the 2->3-bit gap and sits ~1.9x above the TQ3 anchor (0.055) — and buying that requires BOTH a per-channel cache layout AND a sparse side channel.
- **Asymmetric-magnitude ternary is the honest losing rung** (~0.40 KL), as predicted by the doc: three levels discard too much magnitude and, per-channel post-RoPE, it inherits per-channel's smearing problem.
- **Cross-corpus** (gemma3->xval) of the outlier arm degrades +1.5% (within 25%): the outlier mechanism transfers, because a magnitude threshold is not a corpus-specific learned parameter.
- **Samples-per-scale (VQ rule 3).** Per-channel arms B/E fit each scale on >= 365 calib samples (g1) up to 11680 (g32) — all far above the 10-sample memorization line, so their failure is genuine, not undersampling. The one statistic that DOES trip the flag is asym-ternary **D g1**, whose s± means fall to a per-layer minimum of ~3 samples/scale (< 10, flagged); D loses regardless, but its numbers are additionally unreliable for that reason.

### Where E2B actually breaks

NOT the 512-dim globals, at least not offline: global layers reconstruct BETTER than local (2-bit KL global 0.1135 vs local 0.2052); relK is uniform ~0.34 everywhere. The offline per-layer collapse is spread across the many LOCAL sliding-window layers, which each carry ~0.20 2-bit KL. The globals' known fragility is a CLOSED-LOOP phenomenon (they are cache producers reused by many sliding consumers, so their error cascades) that single-layer offline replay cannot see — exactly the isolated-vs-free-running gap the ternary doc flags. This study measures the isolated leg only.

### Verdict: **NO-GO** — E2B 2-bit K is NOT rescued to the operative target.

No deployable arm reaches the TQ3 anchor. The best — `E per-chan mse g1 + 2% outliers` — closes ~62% of the 2->3-bit gap offline and sits ~1.9x above TQ3; after the ~3-5x closed-loop amplification that lands well above the 0.27 cliff, so 2-bit would still collapse in the loop. Crucially, KIVI per-channel ALONE is *net-negative* post-RoPE (a regression vs the per-vector baseline); the sparse outlier channel is the only lever that helps, and although it pays most when combined with per-channel (arm E), realizing that gain needs BOTH a per-channel cache layout AND a sparse side channel.

**Recommendation.** E2B K stays **TQ4 / gated-TQ3**. Do NOT build a per-channel-K cache type or fused per-channel dequant/RoPE kernel for E2B — per-channel alone is a regression post-RoPE, and the extra ~0.033 KL that per-channel+outliers (E) wins over outliers-on-the-current-codec (C/A) does not justify a full cache relayout for a result that is still ~2x TQ3 and closed-loop-unsafe. If any 2-bit work is pursued, the only cost-justified piece is a generic **sparse-outlier side channel** (~1-2% fp16, +0.2-0.4 bpe) bolted onto the CURRENT per-vector Hadamard codec (arm C/A) — a modest, corpus-robust win (cross-corpus loss under 1%) that needs no cache relayout, but does not by itself make 2-bit safe closed-loop.

**Caveat (pre-RoPE untested).** These dumps are POST-RoPE. The ternary doc's central recommendation is PRE-RoPE per-channel quantization, where the outlier-channel structure is intact. This study cannot test that; a pre-RoPE K dump would be needed to falsify the KIVI premise properly. The NO-GO is specifically for post-RoPE per-channel, which is what the cache actually stores today.

## Consolidated E2B 2-bit verdict (2026-07-21) — every lever tested, NO-GO

This study (post-RoPE per-channel + outliers + asym-ternary) plus its siblings close the
E2B 2-bit question exhaustively. All numbers softmax-KL (offline) or Mean KLD (closed-loop).

**Asymmetric K/V, closed-loop** (`llama-perplexity`, K>V hypothesis):

| config | KLD | same-top | note |
|---|---|---|---|
| TQ4/TQ4 | 0.097 | 89.0% | lossless reference |
| TQ4-K / TQ3-V | 0.161 | 85.8% | legit middle rung (9.0 MiB) |
| TQ4-K / TQ2-V | 0.482 | 74.6% | **V is NOT free** — cascades via residual stream |
| TQ3-K / TQ2-V | 0.561 | 72.6% | both degraded |

The single-pass intuition "V doesn't touch rankings so TQ2-V is free" is **refuted
closed-loop**: V error propagates through the layer output → next-layer inputs, so TQ2-V
costs +0.385 KLD even with TQ4-K. V tolerates TQ3 (0.161) but not TQ2.

**Pre-RoPE per-channel** (`docs/leankv-prerope-perchannel-e2b-2026-07.md`, RoPE-reimpl
validation gate passed, cosine 1.000000): pre-RoPE per-channel 0.225 beats post-RoPE
per-channel 0.249 (+9.5% — KVQuant mechanism is REAL) but still loses to the Hadamard
baseline 0.187; best pre-RoPE+2%-outliers 0.098 @ 2.51 bpe, still 1.8× the TQ3 anchor.

**Full lever ledger for E2B 2-bit K (target = TQ3 quality, offline KL 0.055):**

| lever | result | verdict |
|---|---|---|
| per-channel K post-RoPE (KIVI) | 0.249 (loses to 0.187 baseline) | NO — RoPE smears channel structure |
| per-channel K pre-RoPE (KVQuant premise) | 0.225 (mechanism real, still > baseline) | NO — real but not load-bearing |
| dense+sparse outliers | 0.098–0.14 @ ~2.5 bpe (65% gap closed) | **partial win — a MENU item**, not an E2B rescue |
| asymmetric K/V | best 0.161 closed-loop (TQ4-K/TQ3-V), not 2-bit | NO — V also cascades |
| asym-magnitude ternary | 0.394 | NO (as predicted) |
| low-rank projection | earlier NO-GO | NO |
| mixed sub-3.5bpw plans | self-sabotage (tq2 floors) | NO |
| mse_opt scale swap | −7% only (rank-bounded washout) | NO on E2B |

**Conclusion: no method rescues E2B 2-bit K to TQ3 quality. E2B ships on TQ4 /
gated-TQ3, definitively.** The "512-dim globals break" hypothesis was refuted offline
(globals reconstruct *better*: 0.114 vs 0.205 local) — E2B's global fragility is a
closed-loop cache-producer cascade, invisible to single-layer offline replay. The one
transferable positive is a generic **sparse-outlier side-channel** (~65% gap closure,
cross-corpus-stable), which belongs in the adaptive menu, not the E2B slot.
