# VQ Study — product/residual vector quantization of the KV cache (real K/Q dumps)

Generated by `kit-v2/vq_study.py` on 2026-07-19 16:47. Same three KCAL captures, GQA mapping, model-true attention scales, SWA masks and calib/eval split as `contour_study_report.md`.

## Method

- **Fit/eval protocol**: codebooks are k-means-fit on CALIB-half K subvectors only; the **full** K matrix used in eval attention (calib + eval positions) is then encoded/decoded with those frozen codebooks, and all metrics are on EVAL-half queries. Eval-half keys are held out from every fit.
- **Scalar baselines, two regimes**: `u2_half/u3_half` quantize only the calib half (contour-study-identical; used ONLY to verify this pipeline reproduces the quoted contour numbers). `u2_full/u3_full` quantize the full K matrix — the like-for-like baseline for the VQ configs; **gap_closure uses the full-K baselines** so numerator and denominator share one regime.
- **Spaces**: `rot` = per-layer randomized-Hadamard (TurboQuant matrices, seed 42+il), then per-32-element amax block scales stored fp16 — exactly the in-tree TQ2_0/TQ3_0 block scheme (`ggml/src/ggml-tq.c`). `raw` = identical block-scale scheme on unrotated K (VQ may exploit channel correlation that the rotation destroys).
- **Budget accounting (strict)**: effective bpe = code_bits + 0.5 (fp16 scale / 32 elems), the same ladder as scalar TQ (TQ2=2.5, TQ3=3.5). No per-token side info beyond codes + block scales. Codebooks are amortized across all cached tokens; absolute per-layer size = n_pos x codewords x m x 2 bytes is reported per config.
- **k-means**: k-means++ init, fixed seeds, >=25 Lloyd iterations (cap 40, tol 1e-05), empty clusters reseeded to farthest points. **Data limitation**: calib samples per codebook position are tiny — 365 (E2B, n_kv=1), 1460 (gemma3-4b), 3008 (LFM2.5) — far below the 400k subsample cap (no subsampling was needed). Configs where codewords >= samples are flagged DEGEN (codebook memorizes every calib subvector; eval keys get nearest-calib-subvector).
- **SNR** = mean over all T x n_kv key vectors of per-vector 10 log10(||k||^2/||k-k_hat||^2) on the full encoded K (clipped to [-40, 99] dB).

## Runtime control: K-only scalar cache, closed-loop (Gemma4-E2B)

This is the first real-runtime control, not the KCAL replay above. `llama-perplexity`
processed one held-out WikiText-2 block with a 2,048-token context. A separate FP16
run supplied the reference logits. The quantized runs stored **only K** in the stated
TQ format; V remained F16. Later tokens therefore attended through the quantized K
cache in every layer. `LEANKV_NO_QDIM_GATE=1` was required for the TQ2/TQ3 rows,
because the normal safety gate upgrades this model's rank-bounded 256-dim K path to
TQ4.

| K cache | K cache MiB | V cache | mean output KLD | same top token | RMS token-probability change |
|---|---:|---|---:|---:|---:|
| F16 reference | 18.00 | F16 (18.00 MiB) | 0 | 100.00% | 0% |
| TQ4 (automatic safe-gate result) | 5.06 | F16 (18.00 MiB) | 0.06239 | 91.50% | 6.07% |
| TQ3 (gate bypassed) | 3.94 | F16 (18.00 MiB) | 0.16892 | 86.02% | 11.02% |
| TQ2 (gate bypassed) | 2.81 | F16 (18.00 MiB) | 0.90845 | 66.08% | 23.10% |
| raw PQ m=4/256 reference (frozen KCAL codebooks) | 18.00* | F16 (18.00 MiB) | 0.78455 | 67.25% | 22.38% |

**Interpretation:** the runtime amplifies the 2-bit K error strongly: TQ2 changes the
model's top token on 33.92% of evaluated positions, versus 13.98% for TQ3. This is
the closed-loop target for PQ. The first PQ bridge beats TQ2 (0.78455 vs 0.90845
KLD; 67.25% vs 66.08% top-token agreement), but remains far from TQ3. It reconstructs
PQ-decoded K into an F16 cache, so `18.00*` MiB is **not a compression result**. A
packed-PQ cache type and fused attention decode are still required; their quality gate
is to preserve this (or better) runtime result at 2.5 effective bits/element.
